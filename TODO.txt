# ClamFox — Advanced Hardening Roadmap (v0.0.6.x)

This roadmap outlines the next frontier of defensive engineering for ClamFox, focusing on cryptographic integrity, ML-driven heuristics, and zero-knowledge privacy.

### 2. CRYPTOGRAPHIC HARDENING
- [ ] **Intelligence "Time-Lock"**: Require signed timestamps in `urldb.txt` to prevent "Freeze Attacks" (where an attacker prevents updates to keep a stale, vulnerable database in memory).
- [ ] **Zero-Knowledge Whitelisting**: Transition `trust_db.json` to a salted-hash format. The host script should be able to verify a "Member of Trusted Set" proof without ever seeing the cleartext domain name.
- [ ] **Supply-Chain Canaries**: Implement runtime integrity checksums for all loaded `.py` modules. The bridge will re-verify its own code-segment hashes at random intervals to detect "Hot-Patching" or memory injection.

### 3. ADVANCED DEFENSE-IN-DEPTH (Hardening)
- [ ] **Memory-Safe Migration (Rust)**: Rewrite critical Native Messaging and TPM unsealing logic in Rust to eliminate memory-safety vulnerabilities (buffer overflows, use-after-free) inherent in high-performance C/Python glue.
- [ ] **Kernel-Level Integrity (FS-Verity)**: Implement `FS-Verity` or advanced AppArmor profile enforcement to ensure signatures and blocks have not been tampered with by other local processes between check and load.
- [ ] **System-Level Sandboxing**: Transition from standard execution to a restricted `bubblewrap` or `cgroup` container for the native host, enforcing strict "Need-to-Know" access to the filesystem.
- [ ] **WASM Logic Offloading**: Port high-speed matching and homograph logic to WebAssembly. This allows critical checks to run inside the browser's sandbox, reducing the attack surface exposed to the native bridge.
